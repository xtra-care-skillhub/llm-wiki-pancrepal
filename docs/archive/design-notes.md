# LLM-Wiki 医疗框架设计说明（节选）

## Skills 与 Wiki 的关系

Skills 能力与 Wiki 知识库结构需要强关联，不应彼此独立演化。

- Skills 是**能力层**，Wiki 是**知识层**。
- 两者通过统一 `entity_id`、Schema、标签路由（domain/topic/audience）绑定。
- Skills 只读/写标准化卡片与页面，不直接耦合具体渠道文档格式。

## Skills 双向闭环（已确认）

Skills 能力是双向的：既能根据问题提取知识，也能通过对话沉淀知识。

### 1) 读路径（提取知识）
用户问题 → skill 路由 → 检索 wiki/cards → 生成答案 + 引用证据。

### 2) 写路径（沉淀知识）
高价值对话 → skill 结构化提炼 → 写入候选卡片/页面 → 审核或自动发布 → 更新索引。

## 写权限分层建议

1. `read`：所有问答 skills 可用。
2. `propose_write`：仅写入 `draft/`（推荐默认）。
3. `publish`：通过策略门控后写入正式库。

该机制确保：
- 知识库可持续进化；
- 可追溯（来源、版本、变更）；
- 降低噪音与错误直接污染主库的风险。
